# Анализ тональности отзывов

Итоговый проект по курсу Python. Автоматический сбор и анализ тональности отзывов покупателей с маркетплейсов.

## Описание

Проект состоит из двух частей:

1. **Сбор отзывов** — парсинг отзывов с маркетплейсов (Wildberries, Ozon, Яндекс Маркет)
2. **Анализ тональности** — предобработка текста, извлечение ключевых слов и классификация отзывов по тональности (позитивный / нейтральный / негативный)

## Структура проекта

```
├── Sentiment_of_reviews.ipynb          # Jupyter-ноутбук с анализом тональности
├── sentiment_of_reviews.py             # Python-скрипт (экспорт из ноутбука)
├── scrapping_reviews/
│   ├── scrapping_reviews.py            # Основной скрипт парсинга
│   ├── scrapping_reviews_common.py     # Общие утилиты парсинга
│   ├── scrapping_reviews_selenium_all.py   # Парсинг всех маркетплейсов
│   ├── scrapping_reviews_selenium_ozon.py  # Парсинг Ozon
│   ├── scrapping_reviews_selenium_wb.py    # Парсинг Wildberries
│   ├── scrapping_reviews_selenium_ya.py    # Парсинг Яндекс Маркет
│   └── processed_reviews.csv           # Собранные и обработанные отзывы
```

## Файл processed_reviews.csv

Основной датасет, собранный парсером. Содержит **302 отзыва** с трёх маркетплейсов.

### Столбцы

| Столбец | Тип | Описание |
|---|---|---|
| `create_date` | дата | Дата публикации отзыва |
| `name` | строка | Имя автора отзыва |
| `sex` | строка | Пол автора (`мужской` / `женский`) |
| `review` | строка | Текст отзыва (может быть пустым) |
| `rate` | число | Оценка товара от 1.0 до 5.0 |
| `market` | строка | Источник: `Wildberries`, `Ozon`, `Я.Маркет` |

### Использование

Файл служит входными данными для скрипта анализа тональности. При локальном запуске путь указывается напрямую:

```python
df = pd.read_csv('scrapping_reviews/processed_reviews.csv', index_col=0)
```

При запуске в **Google Colab** файл нужно загрузить в директорию `/content/sample_data/`:

```python
df = pd.read_csv('/content/sample_data/processed_reviews.csv', index_col=0)
```

> Перед анализом пустые значения в столбце `review` автоматически удаляются (`dropna`).

## Этапы анализа

1. Чтение и валидация CSV-файла с отзывами
2. Предобработка текста: очистка, токенизация, удаление стоп-слов, лемматизация
3. Построение облака слов до и после очистки
4. Определение ключевых слов и коэффициентов важности (KeyBERT)
5. Классификация тональности каждого отзыва
6. Сохранение результатов в CSV
